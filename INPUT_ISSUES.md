# ❓ 无法在 Quick Calculate 输入数据？

## 问题描述
无法在 Monthly Target 和 Monthly Sales 输入框中输入数据

## 可能的原因和解决方案

### ✅ 原因 1：浏览器缓存
**解决方法：**
1. 完全关闭应用
2. 重新启动
3. 或按 `Ctrl + Shift + R` 强制刷新

### ✅ 原因 2：输入框没有获得焦点
**解决方法：**
1. 用鼠标**点击**输入框
2. 确保看到光标闪烁
3. 然后输入数字

### ✅ 原因 3：输入类型问题
**检查：**
- 输入框期望的是**数字**
- 不要输入逗号或其他符号
- 正确：`50000`
- 错误：`50,000` 或 `RM50000`

### ✅ 原因 4：JavaScript 错误
**检查步骤：**
1. 按 `F12` 打开开发者工具
2. 切换到 "Console" 标签
3. 查看是否有红色错误信息
4. 如果有错误，复制并发送给我

## 🔍 测试输入框是否工作

### 测试方法：
1. 打开 Quick Calculate
2. 点击 "+ Add Salesperson"
3. 尝试点击 "Monthly Target" 输入框
4. 你应该看到：
   - 输入框边框变成蓝色 ✅
   - 光标在输入框内闪烁 ✅
   - 可以输入数字 ✅

### 如果看不到蓝色边框：
- 说明输入框可能有问题
- 打开 F12 查看 Console 错误

## 🆕 新版本改进

我已经在输入框标签旁边添加了 **"✏️ Editable"** 提示，让你知道这个输入框是可以编辑的。

同时：

- 失去焦点时边框变回灰色

## 🔧 如果还是无法输入

### 方法 1：使用 Import Excel 功能
1. 准备好你的 Excel 数据
2. Quick Calculate → Import Excel
3. 系统会自动填充所有数据

### 方法 2：检查是否有其他软件冲突
- 关闭其他 Electron 应用
- 关闭浏览器
- 重启应用

### 方法 3：重新安装
```bash
# 删除配置文件
# Windows: %APPDATA%/sales-commission-calculator
# macOS: ~/Library/Application Support/sales-commission-calculator
# Linux: ~/.config/sales-commission-calculator

# 重新安装依赖
npm install

# 启动
npm start
```
